Архитектура Java-кода (src/main/java/net/wizard/elemental_awakening/)ElementalAwakening.java — Главный класс мода. Регистрирует предмет Посох Земли (earth_staff) через DeferredRegister<Item>. В конструкторе запускает регистрацию сети ModMessages.register();.mana/PlayerMana.java — Контейнер данных маны игрока. Переменные: mana (дефолт 100), maxMana (100). Методы: consumeMana(float), regenMana() (восстанавливает 0.2F за тик, то есть 4 маны в секунду), setMana(float).mana/PlayerManaProvider.java — Провайдер Капабилити (Capability) маны. Привязывает данные к игроку, сериализует и десериализует их в NBT ("mana") для сохранения маны при перезаходе в мир или смерти.networking/ManaSyncDataPacket.java — Сетевой пакет для синхронизации маны. Считывает float из буфера, на клиенте находит Minecraft.getInstance().player и через капабилити принудительно обновляет ему ману, чтобы полоска двигалась.networking/ModMessages.java — Создает сетевой канал SimpleChannel с именем messages. Регистрирует пакет ManaSyncDataPacket в направлении PLAY_TO_CLIENT. Содержит метод sendToPlayer(MSG, ServerPlayer).client/ManaHudOverlay.java — Клиентский HUD. Отрендерен по центру экрана (height - 55). Отрисовывает подложку mana_empty.png целиком (192х6), считывает ману игрока через капабилити, высчитывает пропорцию в пикселях и поверх накладывает mana_full.png, обрезая её по ширине.client/EarthMenuScreen.java — Кастомный класс экрана Screen (без использования дефолтных кнопок Майнкрафта, чтобы не ломался при обновлениях Forge). Рендерит фон earth_menu.png. Методы render и mouseClicked вручную просчитывают математические координаты квадрата 40х40 в центре плиты. Если мышка в координатах — рисует stena_hover.png, если нет — stena.png. При клике ЛКМ выводит сообщение в чат и закрывает меню.event/ModEvents.java — Серверный обработчик событий (@Mod.EventBusSubscriber).onAttachCapabilitiesPlayer — привязывает PlayerManaProvider к сущности игрока.onPlayerTick — на стороне сервера (!level.isClientSide) каждый тик вызывает mana.regenMana() и отправляет пакет синхронизации ModMessages.sendToPlayer на клиент.onLevelTick — тик мира. Пробегается по списку EarthStaffItem.ACTIVE_WALLS, вызывает у каждой активной стены метод wall.tick(), а завершенные удаляет из списка.event/ClientModEvents.java — Клиентский обработчик событий (bus = Bus.MOD, value = Dist.CLIENT). В событии RegisterGuiOverlaysEvent регистрирует наш оверлей маны выше полоски здоровья: event.registerAbove(VanillaGuiOverlay.PLAYER_HEALTH.id(), "mana_bar", ManaHudOverlay.HUD_MANA);.item/EarthStaffItem.java — Логика Посоха Земли (EarthStaffItem extends Item).В методе use: на клиенте при player.isCrouching() (Shift+ПКМ) открывает меню Minecraft.getInstance().setScreen(new EarthMenuScreen());.На сервере при обычном ПКМ проверяет капабилити маны. Если >= 10.0F, списывает 10 маны. Вычисляет координаты в 1 блоке перед игроком (player.blockPosition().relative(direction, 1)) и строит огромную стену 5 в ширину и 3 в высоту (15 блоков грязи), центрируя её перед глазами игрока.Внутри класса объявлен статический список активных стен public static final List<ActiveWall> ACTIVE_WALLS = new ArrayList<>(); и внутренний класс ActiveWall.ActiveWall хранит три ряда списков BlockPos для постепенного роста стены. В момент спавна блоков метод spawnRow запоминает оригинальный BlockState блоков мира, которые стояли на этих координатах (камень, доски и т.д.), и записывает их в свои списки savedPositions и savedStates.Метод tick() стены вызывается из тика мира. На 10-й тик (0.5 сек) спавнится 2-й ряд блоков, на 20-й тик (1 сек) — 3-й ряд. Спавн блоков сопровождается серверными частицами level.sendParticles (частицы грязи) и звуками SoundEvents.GRAVEL_PLACE в главном потоке.На 220-й тик (10 секунд удержания стены + 1 секунда роста) вызывается метод restoreWorld(). Он пробегается по сохраненным координатам и, если там всё еще стоит грязь, полностью восстанавливает оригинальные блоки мира назад, не оставляя дыр. Восстановление сопровождается сочным взрывом частиц разрушения грязи и звуком SoundEvents.GRAVEL_BREAK.🎯 ЧТО МЫ БУДЕМ ДЕЛАТЬ ДАЛЬШЕ:Сейчас мы зафиксировали этот рабочий код и готовы двигаться дальше по нашему древу прокачки Земли. 👑 ГЛОБАЛЬНАЯ КОНЦЕПЦИЯ МОДА "ELEMENTAL AWAKENING"1. Система Магических РанговКаждый игрок начинает свой путь обычным человеком без магии. Используя заклинания стихий, он прокачивает свой уровень и повышает Ранг Мага:Ранг E (Новичок): Доступна только простейшая магия защиты (наша текущая стена земли «Барьер Скаута I»). Заклинания слабые, стоят мало маны.Ранг D (Ученик): Открываются первые атакующие техники (Земляной Шип). Мана регенерирует чуть быстрее.Ранг C (Мастер): Появляются массовые заклинания контроля (Земляное Трясение, Огненный Шторм).Ранг B (Аристократ): Мощнейшая магия изменения ландшафта и призыва элементалей.Ранг A / S (Высший маг / Легенда): Ультимативные способности, способные уничтожать целые данжи и боссов за один каст.🌟 2. Древо Прокачки Стихий (RPG-Меню)В нашем меню (каменная плита earth_menu.png) способности не открыты сразу. Это ветвистое древо сверху вниз:Игрок обязан зарабатывать Магический опыт (за убийство мобов магией или копание руд).Сверху дерева находится наша Стена (Ранг Е). Покупаешь её — от неё вниз в меню открываются стрелочки к следующим способностям Земли Ранга D и C.Пока не куплена предыдущая магия, нижняя кнопка заблокирована (горит серым или вообще скрыта).🔥 3. Четыре СтихииВ будущем мы хотели добавить вкладки или отдельные посохи для остальных элементов:Земля (Текущая): Защита, создание барьеров, замуровывание врагов в блоках, призыв каменных шипов.Огонь: Чистый урон, поджог по площади, огненные шары, взрывы.Вода: Исцеление, заморозка врагов в лед, водяные хлысты.Воздух: Невероятная мобильность, высокие прыжки, полеты, откидывание врагов ветром.🧙‍♂️ 4. Как это должно ощущаться (Аниме-стиль)Магия не должна быть скучным кликом мышки. При касте заклинаний высокого ранга на экране должны лететь сочные спецэффекты, вокруг игрока должны закручиваться магические круги из частиц, а звуки должны быть громкими и эпичными (как в крутых аниме про магов), чтобы каждый чувствовал себя непобедимым Аватаром.Вот этот глобальный план мы и заложили, когда писали первую стену!
